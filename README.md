Проект по разработке сквозной интерактивной аналитики для мониторинга коммерческих показателей кофейных автоматов на базе облачной BI-платформы **Yandex DataLens**.

## 🎯 1. Бизнес-контекст и задача

Сеть автоматизированных кофе-точек непрерывно фиксирует детальные транзакции (дата, тип оплаты, название напитка, сумма). Департамент планирования формирует целевые показатели по выручке в разрезе каждого напитка с шагом в один месяц.

**Цель проекта:** Спроектировать и развернуть отказоустойчивую модель данных и интерактивный дашборд, который позволит руководству компании:
1. Контролировать ключевой KPI — **процент выполнения плана по выручке**.
2. Оценивать общую динамику продаж в годовом разрезе.
3. Анализировать долевую структуру популярности напитков для оптимизации логистики ингредиентов.

## 🛠 2. Технический вызов и архитектурное решение

Главная техническая сложность проекта заключалась в **несовпадении гранулярности (детализации) данных**:
* **Таблица Факт** содержит миллионы строк с ежедневными транзакциями.
* **Таблица План** имеет агрегированный вид — строго 1 строчка на каждый напиток в месяц (датирована 1-м числом).

### Риски классического объединения (JOIN):
* **Прямое объединение** по полю точной даты (`date = plan_date`) привело бы к потере 97% данных факта, так как транзакции за 2-31 числа месяца не нашли бы совпадений в плане.
* **Объединение только по названию напитка** вызвало бы проблему **fan-out (дублирование)**, искусственно размножив ежемесячный план на каждый день продаж (завышение плана в 30 раз).

### Реализованное решение:

1. На этапе предобработки данных создано техническое поле временной привязки (`link_date`), округляющее любую дату транзакции до 1-го числа соответствующего месяца.
2. Для предотвращения дублирования данных на уровне BI-системы внедрено продвинутое **LOD-выражение (Level of Detail)** с фиксированным контекстом агрегации. Метрика «Выручка План» рассчитывается по формуле:

```text
SUM(AVG([plan_revenue] FIXED [plan_date], [coffee_name]))

💡 Логика формулы: BI-система сначала изолирует дубликаты плана внутри конкретного месяца для каждого напитка, вычисляя среднее (AVG), а затем корректно суммирует чистые показатели на графиках любого масштаба.

📊 3. Структура дашборда и ключевые инсайты
На основе построенной модели данных развёрнуты три взаимосвязанных аналитических слоя:

KPI-секция (Индикатор): Верхнеуровневый баннер, отображающий интегральный процент выполнения плана.

Динамика План-Факт (Столбчатая диаграмма): Помесячное сопоставление целевых и реальных показателей выручки.

Структура продаж (Круговая диаграмма): Доли напитков в формировании общего финансового потока.

Ключевые инсайты из данных:
Главные драйверы выручки: Самыми прибыльными позициями являются Latte (27,8K ₽) и Americano with Milk (25,3K ₽). В сумме они генерируют почти половину кассы автомата.

Пиковые периоды: Самым успешным месяцем стал Октябрь 2024 года, где план был перевыполнен на 101,25% (выручка составила 13 891,16 ₽).

Точки роста: Наибольшее проседание относительно плана зафиксировано в Сентябре 2024 года (выполнение всего 94,89%). Этот период требует дополнительного исследования (возможные технические простои автоматов или сбои поставок).

Дашборд полностью интерактивен. Сверху установлен управляющий Селектор (Фильтр) по напиткам:

Вы можете выбрать один или несколько конкретных видов кофе (например, только Cappuccino и Espresso).
